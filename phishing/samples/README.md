# Samples for debugging false positives

Place JSON/CSV sample feature vectors captured from the app here (e.g., `naver_sample.json`) for analysis. Collected samples should include:

- `RAW_FEATURES_JSON` (raw JS payload logged by the app),
- the parsed `features` map (types/values after parsing),
- timestamp and URL metadata.

Example filename: `phishing/samples/naver_20251121_123456.json`
